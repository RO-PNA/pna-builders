# v0.3 — Events (Offline Programs & Gatherings)

> **Version**: v0.3 / **Status**: Draft / **Depends on**: v0.2.1 (Auth hardening) / **Next**: v0.4 (Knowledge Graph)

PNA의 오프라인 활동(컨퍼런스, 워크샵, 정기모임, 스터디)을 **발견·신청·아카이빙**할 수 있는 허브 페이지를 만든다. 현재 워크샵 챗봇(`/26-1q-workshop` = AX Lab)은 단일 이벤트에 특화된 인터랙티브 도구인 반면, Events 페이지는 **전체 프로그램을 시간순으로 조감**하는 역할이다.

---

## 1. 배경 & 문제 정의

### 현재 상태 (v0.2)
- 오프라인 활동 정보가 **카카오톡 공지/구글폼 링크**로만 전파됨
- 지난 행사 기록이 남지 않음 — 참가하지 못한 사람은 내용을 알 수 없음
- 행사 후기가 개인 블로그/슬랙에 흩어져 있고 PNA 플랫폼에 축적되지 않음
- GNB에 워크샵 1개만 하드코딩 — 행사 규모 확장 시 확장 불가

### 문제
- **발견성**: 어떤 행사가 예정돼 있는지, 과거에 뭘 했는지 한눈에 보이지 않음
- **홍보**: 외부 유입 경로(SEO/공유)가 없음 — 행사별 고유 URL 부재
- **아카이빙**: 행사 콘텐츠(발표 자료, 후기, 사진)가 플랫폼에 쌓이지 않음
- **커뮤니티 신뢰**: "이 커뮤니티가 실제로 활동한다"는 인상을 신규 방문자에게 주기 어려움

### 목표
**"PNA가 뭘 하는 곳인지, 다음엔 뭘 하는지, 지금까지 뭘 했는지를 한 페이지에서 보여준다."**

---

## 2. 범위 (Scope)

### In-Scope
1. **`/events` 메인 페이지** — 히어로 배너, 예정 행사, 지난 행사, 카테고리 필터
2. **`/events/[slug]` 상세 페이지** — 행사 정보, 신청 CTA, 갤러리, 후기 섹션
3. **행사 카테고리** — conference, workshop, meetup, study, etc.
4. **후기·리캡** — 행사 종료 후 사진·요약·참가자 후기 표시
5. **관리자 등록** — Supabase 직접 입력 또는 `/admin/events` 최소 폼 (v0.2.1 admin 권한 활용)

### Out-of-Scope (v0.4+)
- 결제·유료 티켓 연동 (토스/카카오페이)
- 참가 인원 실시간 집계
- 이메일·알림 자동 발송
- 캘린더 구독(ICS)
- AX Lab 연동 (현재 별도 라우트 유지)

## 5. 데이터 모델

```sql
-- 행사
create table public.events (
  id bigint generated by default as identity primary key,
  slug text unique not null,
  title text not null,
  category text not null check (category in (
    'conference','workshop','meetup','study','social'
  )),
  description text,             -- 마크다운 본문
  summary text,                 -- 한줄 요약 (카드/OG용)
  thumbnail_url text,           -- 썸네일 이미지
  event_date timestamptz,       -- 행사 시작일시
  event_end_date timestamptz,   -- 행사 종료일시 (선택)
  location text,                -- 장소
  capacity int,                 -- 정원 (null = 제한 없음)
  is_free boolean default true,
  price text,                   -- 유료 시 가격 표기 (예: "3만원")
  registration_url text,        -- 외부 신청 링크 (구글폼 등)
  status text not null default 'draft' check (status in (
    'draft','upcoming','ongoing','ended','cancelled'
  )),
  is_featured boolean default false,  -- 히어로 배너 노출 여부
  gallery_urls text[],          -- 행사 후 사진 URL 배열
  created_by uuid references auth.users(id),
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- 행사 후기
create table public.event_reviews (
  id bigint generated by default as identity primary key,
  event_id bigint not null references public.events(id) on delete cascade,
  user_id uuid not null references auth.users(id),
  author_name text not null,
  content text not null,
  rating int check (rating between 1 and 5),  -- 선택적 별점
  created_at timestamptz default now()
);

-- RLS
alter table public.events enable row level security;
create policy "Events viewable by everyone" on public.events for select using (true);
create policy "Admins can manage events" on public.events for all
  to authenticated using (
    exists (select 1 from public.profiles where id = auth.uid() and is_admin = true)
  );

alter table public.event_reviews enable row level security;
create policy "Reviews viewable by everyone" on public.event_reviews for select using (true);
create policy "Authenticated users can write reviews" on public.event_reviews
  for insert to authenticated with check (auth.uid() = user_id);
create policy "Users can delete own reviews" on public.event_reviews
  for delete to authenticated using (auth.uid() = user_id);
```

## 6. API

| 엔드포인트 | 기능 |
|------------|------|
| `GET /events` (서버 컴포넌트) | `?category`, `?status` 필터. upcoming → ended 순 정렬 |
| `GET /events/[slug]` (서버 컴포넌트) | 행사 상세 + 후기 목록 |
| `POST /api/events` | 행사 등록 (관리자) |
| `PATCH /api/events/[id]` | 행사 수정 — status, gallery 등 (관리자) |
| `POST /api/event-reviews` | 후기 작성 (로그인 유저) |
| `DELETE /api/event-reviews/[id]` | 본인 후기 삭제 |

## 7. 행사 상태 라이프사이클

```
draft → upcoming → ongoing → ended
                 ↘ cancelled
```

| 상태 | 표시 | 비고 |
|------|------|------|
| `draft` | 비노출 | 관리자만 접근 |
| `upcoming` | "모집 중" / "마감 임박" | 날짜 기반 자동 배지 |
| `ongoing` | "진행 중" | `event_date ≤ now < event_end_date` |
| `ended` | "종료됨" + 갤러리·후기 오픈 | `now ≥ event_end_date` |
| `cancelled` | "취소됨" | 별도 안내 문구 |

> **자동 전환**: 서버 컴포넌트 렌더 시 날짜 비교로 UI 배지를 동적 결정. DB status는 관리자가 수동 전환하되, upcoming→ended는 cron 또는 렌더 시점 자동 판단.

## 9. 단계별 릴리즈

| 단계 | 내용 | 기준 |
|------|------|------|
| Step 1 | 스키마 + 시드 (기존 워크샵 1건 등록) | `events` 테이블 배포 |
| Step 2 | `/events` 메인 (배너 + 예정/지난 행사) | 정적 렌더 동작 |
| Step 3 | `/events/[slug]` 상세 + OG 메타 | 외부 공유 미리보기 |
| Step 4 | 후기 기능 (`event_reviews`) | 로그인 유저 후기 작성·삭제 |
| Step 5 | 관리자 등록 UI (`/admin/events`) | 행사 CRUD |
| Step 6 | 갤러리 (사진 업로드/표시) | Supabase Storage 연동 |

## 10. 성공 지표

- 행사 등록 후 **24시간 내 페이지 뷰 ≥ 50** (SEO/공유 기반 유입)
- 행사 상세 → 신청 링크 **CTR ≥ 30%**
- 행사 종료 후 **후기 작성률 ≥ 20%** (참가자 대비)
- `/events` 재방문율 — 주간 **≥ 15%** (다음 행사 확인 동기)

## 12. 메뉴 이름 대안 검토

| 후보 | 장점 | 단점 |
|------|------|------|
| **events** | 직관적, 글로벌 표준 | 프로모션/할인 느낌이 들 수 있음 |
| **programs** | 교육·커뮤니티 프로그램 뉘앙스, 체계적 인상 | 좀 딱딱함 |
| **gather** | PNA "함께" 브랜딩과 부합, 따뜻한 톤 | 의미가 즉시 와닿지 않을 수 있음 |
| **meetups** | 커뮤니티 네트워킹 강조 | 컨퍼런스·스터디까지 아우르기 좁음 |
| **calendar** | 일정 중심임을 명확히 전달 | 캘린더 뷰를 기대하게 됨 |
| **sessions** | 워크샵·스터디와 잘 어울림 | 로그인 세션과 혼동 |
| **connect** | 네트워킹 의미, PNA 브랜드 부합 | 사람 연결인지 행사인지 모호 |

**추천 순위**: `events` (가장 안전) > `programs` (체계적 인상) > `gather` (브랜드 차별화)

events가 가장 무난하고, 브랜드 톤을 살리고 싶으면 `gather`도 괜찮습니다. Knowledge / AX Lab 등 영문 메뉴와 병행 시 `programs`가 톤 밸런스가 좋습니다.
