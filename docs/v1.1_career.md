# v1.1 — Career (커리어 성장 플랫폼)

> **Version**: v1.1 / **Status**: Draft / **Depends on**: v1.0 (정식 출시) / **Next**: v1.2 (디지털 명함)

동문 네트워크의 신뢰를 기반으로 **추천 채용·이직 스터디·커피챗**을 하나의 커리어 허브에서 제공한다.

---

## 1. 배경 & 문제 정의

### 현재 상태
- 이직 스터디가 Events 내 단발 프로그램으로만 존재 (퍼스트펭귄 이직 스터디)
- 추천 채용은 카카오톡 오픈채팅에서 비공식적으로 이루어짐
- 커피챗은 DM 기반 — 매칭·트래킹 불가

### 문제
- 동문 간 추천 채용이 체계적으로 이루어지지 않음
- 이직 스터디의 커리큘럼·과제·환급 관리가 수동
- "이 사람에게 커리어 조언을 받고 싶다"는 니즈가 있지만 진입 장벽이 높음

### 목표
**"PNA 멤버라면 커리어 고민을 나누고, 기회를 발견하고, 함께 준비할 수 있는 허브"**

---

## 2. 범위 (Scope)

### In-Scope
1. **`/career` 메인 페이지** — 추천 채용 보드 + 진행 중 스터디 + 커피챗
2. **추천 채용 보드** — 동문이 올리는 채용 공고 + 레퍼럴 연결
3. **이직 스터디 관리** — 기수별 모집·참가자·커리큘럼·과제·환급 트래킹
4. **커피챗 매칭** — 직무·목표 기반 1:1 매칭 요청

### Out-of-Scope (v2.0+)
- 외부 채용 플랫폼 연동 (원티드/LinkedIn 크롤링)
- AI 기반 이력서 자동 검토
- 멘토링 프로그램 (장기 1:N)
- 결제 시스템 (스터디 참가비)

## 4. 데이터 모델 (초안)

```sql
-- 추천 채용
create table public.job_referrals (
  id bigint generated by default as identity primary key,
  title text not null,
  company text not null,
  position_type text, -- fulltime / contract / intern
  description text,
  referrer_id uuid references auth.users(id),
  referrer_name text,
  apply_url text,
  deadline date,
  status text default 'open' check (status in ('open','closed','filled')),
  created_at timestamptz default now()
);

-- 이직 스터디 기수
create table public.career_studies (
  id bigint generated by default as identity primary key,
  title text not null,
  season int not null, -- 기수
  status text default 'recruiting' check (status in ('recruiting','ongoing','completed')),
  start_date date,
  end_date date,
  capacity int default 10,
  price int,
  created_at timestamptz default now()
);

-- 커피챗 요청
create table public.coffee_chats (
  id bigint generated by default as identity primary key,
  requester_id uuid references auth.users(id),
  target_id uuid references auth.users(id),
  topic text,
  status text default 'pending' check (status in ('pending','accepted','declined','completed')),
  created_at timestamptz default now()
);
```

## 5. 오픈 이슈

- [ ] 추천 채용 보드의 노출 범위 — PNA 멤버 전용 vs 공개
- [ ] 레퍼럴 요청 시 개인정보 전달 방식 — 플랫폼 내 메시징 vs 이메일

---

## 6. 성공 지표

- 월간 추천 채용 게시 **≥ 5건**
- 레퍼럴 연결 성사율 **≥ 20%**
- 이직 스터디 기수당 완주율 **≥ 70%**
- 커피챗 월간 매칭 **≥ 10건**
