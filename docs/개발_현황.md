# PNA 개발 현황

> 현재 개발된 기능·페이지·인프라를 **실제 구현 상태 그대로** 추적한다.
> 새 기능이 머지되면 해당 섹션을 갱신한다. 계획·미구현 항목은 `서비스_로드맵.md`로 분리.
> **현재 버전**: v0.2 (개발 중) / **마지막 업데이트**: 2026-04-16

## 1. 배포·인프라

| 항목 | 상태 |
|------|------|
| 호스팅 | Vercel (`pna-builders.vercel.app`) |
| 프레임워크 | Next.js 16 + Turbopack |
| React | 19 |
| 스타일 | Tailwind CSS v4 + `@tailwindcss/typography` |

| 마크다운 렌더링 | `streamdown` (AI 응답용) |
| 기타 라이브러리 | `react-nice-avatar`, `recharts`, `@supabase/ssr` |

### Supabase 스키마 (확인된 테이블)
- `items` — 콘텐츠 아카이브 (title, url, summary, author, category, points, parent_id, created_by)
- `comments` — 아이템 댓글 (item_id, user_id, author_name, content)
- `workshop_sessions` — 워크샵 팀 세션 (team_name, domain_key, current_stage, status)
- `workshop_messages` — 워크샵 대화 로그

## 2. 인증 — 구글 로그인 (v0.2.1 코드 작업 완료)

### 2-1. 구현 범위
- Provider: **Google OAuth 단일** (Supabase 경유, PKCE 플로우)
- 로그인 진입: `src/components/LoginButton.tsx` — 현재 pathname을 `next` 파라미터로 부착 후 `/auth/callback`로 리다이렉트
- 로그아웃: `src/components/LogoutButton.tsx` — `supabase.auth.signOut()` → `router.refresh()`
- OAuth 콜백: `src/app/auth/callback/route.ts` — `exchangeCodeForSession` + 에러 핸들링 + `next` 복귀
- 세션 갱신: `src/proxy.ts` + `src/utils/supabase/middleware.ts` — Next.js 16 proxy 컨벤션으로 모든 요청에서 세션 자동 갱신
- 서버 컴포넌트: `src/utils/supabase/server.ts` — `createServerClient` + `getAll/setAll` 쿠키 핸들러
- 클라이언트: `src/utils/supabase/client.ts` — `createBrowserClient`
- 관리자 권한: `src/utils/supabase/admin.ts` — Service Role Key, 세션 미유지

### 2-2. 인증 플로우
```
1. 유저가 "login" 클릭 → signInWithOAuth({
     redirectTo: `${origin}/auth/callback?next=${현재 경로}`
   })
2. Supabase → Google OAuth 동의 → Supabase 콜백
3. /auth/callback?code=<code>&next=<경로>로 리다이렉트
4. route handler가 exchangeCodeForSession(code) 호출 → 쿠키에 세션 저장
5. next 경로로 최종 리다이렉트
6. 이후 요청은 proxy.ts가 매 요청마다 getUser() 호출 → 세션 자동 갱신
```

### 2-3. 잔여 과제 (v0.2.1 사용자 설정 대기)
- Supabase Dashboard Auth URL 설정 (Site URL, Redirect URLs)
- `supabase/migrations/v0.2.1_profiles.sql` 실행 (`profiles` 테이블 + 트리거)
- 본인 계정 `is_admin=true` 승격
- 브라우저 실측 테스트
- → 전체 체크리스트: `docs/v0.2.1_google_oauth_debugging.md` §9

### 2-4. 현재 로그인 후 얻는 것
- 네브바에 이메일 prefix 표시
- `submit` 메뉴 노출 (네브바 + 홈 상단 버튼)
- `/api/items` POST 허용 (RLS: authenticated insert)
- `/api/comments` POST로 댓글 작성

### 2-5. 후속 (P1)
- 로그인 유도 UX(모달/배너), `/profile` 편집, 네브바 아바타, `auth_error` 쿼리 안내

## 3. 페이지별 구현 현황

### `/` — Knowledge 홈
- **구현 완료**
- Supabase `items` 테이블에서 최신순 목록 렌더
- 오른쪽 사이드바: 최신 댓글 10개 (lg 이상에서만 노출)
- 비로그인 상태에서도 열람 가능
- submit 버튼(헤더/상단)은 로그인 필요

### `/item/[id]` — 아이템 상세
- **구현 완료**
- Supabase에서 단일 아이템 조회
- OG 메타데이터 (title, description, url, publishedTime) 자동 생성
- 댓글 섹션 (`CommentSection`)
- 마크다운 복사 버튼 (`CopyMarkdownButton`)
- `revalidate = 0` (즉시 반영)

### `/submit` — 아이템 등록
- **구현 완료**
- 인증 필요 (비로그인 시 alert 후 차단)
- URL / summary / category 입력
- `/api/items`로 POST

### `/about` — PNA 소개
- **구현 완료**
- 갤러리 3장, Why PNA / What We Do / Join Us 섹션 (정적)

### `/open-chat` — 대학별 오픈채팅 허브
- **구현 완료** (정적 하드코딩)
- 서울시립대(105/200), 한국외대(17/100), 전북대(8/100)

### `/26-1q-workshop` — 오프라인 워크샵 챗봇
- **구현 완료** (1,300줄)
- 팀 등록: 팀명 → 팀원 + 프레임워크 성향(SCQA/MECE/TDCC/5-Whys/XYZ) → 도메인
- 4개 도메인: 페이플로(핀테크), 프레시마켓(커머스), 클립비(숏폼), 애드포커스(B2B SaaS)
- 도메인별 지표, 6개월 트렌드, 3명의 VIP
- 6단계 Phase (P2↔P4 순서 교체 반영): 문제 감지 → 원인 추적 → 문제 분해 → 문제 탐색 → 가설 검증 → 종합
- Gemini 스트리밍 응답
- VIP 만족도 실시간 파싱 + 펭귄 아바타 표시
- recharts 기반 VIP 트렌드 차트
- Phase별 타이머
- 세션 영속성 (localStorage + Supabase `workshop_sessions/workshop_messages`)
- 클리어 연출(confetti), 운영자용 시스템 모달(비밀번호)
- Knowledge RAG (items 테이블 기반 관련 자료 주입)
- 모바일 반응형

### `/26-1q-workshop/framework-test` — 프레임워크 유형 테스트
- **구현 완료** (464줄)
- 7개 상황 × 5지선다 → 성향 진단
- 5개 프레임워크 매핑: 🔍 TDCC / 🎯 S-C-Q-A / 📐 MECE+LogicTree / 🔗 5-Whys / 🧪 XYZ 가설

## 갱신 규칙

- 페이지·API·컴포넌트가 추가/삭제되면 해당 섹션을 즉시 수정
- 의미 있는 머지 후 "마지막 업데이트" 날짜와 "최근 주요 변경 이력" 갱신
- 로드맵(예정 작업)은 절대 이 문서에 쓰지 않는다 → `서비스_로드맵.md`로
- 미구현 기능은 섹션에 "미구현"으로 표기하거나 제거
