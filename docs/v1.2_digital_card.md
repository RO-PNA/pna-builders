# v1.2 — 디지털 명함 (Digital Card)

> **Version**: v1.2 / **Status**: Draft / **Depends on**: v1.1 (Career) / **Next**: v2.0+

PNA 멤버의 정체성을 **카드 한 장**으로 표현하고, 오프라인(NFC 명함)과 온라인(프로필 페이지)을 연결한다.

---

## 1. 배경 & 문제 정의

### 현재 상태
- `profiles` 테이블에 `display_name`, `avatar_url`, `is_admin`만 존재
- 네브바에 이메일 prefix만 표시
- 오프라인 행사에서 서로를 기억할 수단이 없음 — 포스트잇 이름표에 의존
- NFC 명함/포토카드 구상은 있으나 디지털 연동 없음

### 문제
- "이 사람이 어떤 일을 하는 사람인지" 한눈에 볼 수 없음
- 오프라인 행사 후 온라인에서 연결이 끊김
- 동문 네트워크의 핵심 가치인 "서로를 발견"할 수단 부재

### 목표
**"PNA 멤버라면 누구나 자신만의 디지털 명함을 갖고, NFC 한 번으로 연결된다"**

---

## 2. 범위 (Scope)

### In-Scope
1. **`/profile` 편집 페이지** — 이름·직무·회사·경력·자기소개·관심사·포트폴리오 URL
2. **`/member/[slug]` 공개 프로필** — 디지털 명함 카드 뷰 + 공유 링크
3. **펭귄 캐릭터 커스터마이저** — 아바타 생성 (react-nice-avatar 확장 또는 커스텀)
4. **명함 카드 디자인** — 카드 형태 UI, OG 이미지 자동 생성, 캡처/공유
5. **NFC 연동 설계** — NFC 태그 → 공개 프로필 URL 매핑
6. **멤버 디렉토리** — `/members` 전체 멤버 목록 (직무·대학 필터)

### Out-of-Scope (v2.0+)
- 실물 NFC 명함/포토카드 제작·배송
- QR코드 기반 명함 교환
- 경력 인증 (LinkedIn 연동)
- 포트폴리오 호스팅

## 4. 데이터 모델 (초안)

```sql
-- profiles 테이블 확장
alter table public.profiles add column if not exists slug text unique;
alter table public.profiles add column if not exists job_title text;
alter table public.profiles add column if not exists company text;
alter table public.profiles add column if not exists years_of_experience int;
alter table public.profiles add column if not exists university text;
alter table public.profiles add column if not exists bio text;
alter table public.profiles add column if not exists interests text[];
alter table public.profiles add column if not exists portfolio_url text;
alter table public.profiles add column if not exists avatar_config jsonb; -- 펭귄 커스터마이저 설정
alter table public.profiles add column if not exists is_public boolean default false;

-- NFC 매핑
create table if not exists public.nfc_cards (
  id bigint generated by default as identity primary key,
  card_uid text unique not null,  -- NFC 태그 UID
  user_id uuid references auth.users(id),
  activated_at timestamptz,
  created_at timestamptz default now()
);
```

## 5. NFC 연동 설계

```
[NFC 태그 터치]
  → 브라우저 열림: https://pna.community/nfc/[card_uid]
    → /nfc/[card_uid] 라우트가 DB에서 user_id 조회
      → /member/[slug] 공개 프로필로 리다이렉트
```

- NFC 태그에는 고정 URL만 기록 (한 번 기록 후 변경 불필요)
- 미등록 카드 터치 시 → 카드 등록 페이지로 안내
- 실물 제작은 v2.0+ (디자인·인쇄·배송)

---

## 6. 펭귄 캐릭터 커스터마이저

기존 `react-nice-avatar`를 활용하되, PNA 브랜드에 맞는 커스텀 옵션 추가:
- 배경색 (브랜드 오렌지 / 사용자 선택)
- 액세서리 (동문 대학 마크, 직무 아이콘)
- 포즈 (프로필 / 명함 / 이벤트용)

설정값은 `profiles.avatar_config` (JSONB)에 저장.

---

## 7. OG 이미지 자동 생성

공개 프로필 URL 공유 시 명함 카드가 미리보기로 표시되도록:
- `@vercel/og` 또는 Satori 기반 동적 OG 이미지 생성
- `/api/og/member/[slug]` → 명함 디자인의 PNG 렌더
- 카카오톡·슬랙·링크드인 공유 시 자동 미리보기

## 8. 오픈 이슈

- [ ] 프로필 공개 범위 — 기본 비공개 vs 기본 공개
- [ ] 펭귄 커스터마이저 수준 — react-nice-avatar 그대로 vs 커스텀 일러스트
- [ ] NFC 카드 물리적 규격 — 표준 명함 크기 vs 포토카드 크기
- [ ] NFC 카드 비용·제작 파트너
- [ ] 멤버 디렉토리 노출 범위 — 로그인 유저만 vs 전체 공개
- [ ] `profiles.slug` 자동 생성 규칙 (이름 기반 vs 사용자 지정)

---

## 9. 성공 지표

- 프로필 완성률 (필수 필드 채운 멤버 비율) **≥ 60%**
- 공개 프로필 월간 조회수 **≥ 100**
- 명함 카드 공유 횟수 **≥ 30/월**
- NFC 카드 활성화율 (발급 대비) **≥ 50%** (실물 제작 후)
